Укажите, какие поля профиля контракта изменяются в соответствии с новыми приложением

При загрузке страницы пользователь видит 4 закрытых блока (вид при загрузке ниже). Для просмотра и изменения условий пользователь нажимает кнопку "+". При нажатии каждой кнопки "+", открываются поля соответствующего блока, при этом кнопка "+" меняется на кнопку "-", нажатием которой поля блока можно закрыть. На макете показаны 2 кнопки через / для информации.

Полностью открытые блоки и работа с ними изображена после метки "Блоки в открытом состоянии".

По каждому полу нужно определиться, может ли пользователь его корректировать с помощью нового приложения к контракту. Для удобства (пока этой определенности нет) возможность изменения показана для всех полей, кроме "Тип контракта", "Указать свой вариант", "Номер контракта", "Дата начала". До согласования приложения на страницы видны оба поля, после приобретения приложением статуса "Верифицирован", все поля профиля материнского контракта приобретают итоговые значения, изложенные в требованиях. Все поляпрофиля приложения остаются теми, которые введены пользователем.

Все поля, которые доступны для редактирования, меняются по следующему принципу: если приложение меняет содержание поля, пользователь нажимает кнопку "Изменить", расположенную возле этого поля, и рядом с материнским полем раскарывается поле (или блок), аналогичный тому, который требуется изменить, и подсказка для клиента: "Введите значение согласно приложению к контракту" и в открывшемся поле вводит значение поля согласно приложению. Это изображено на примере даты окончания, в дальнейшем изображены только кнопки "Изменить".
