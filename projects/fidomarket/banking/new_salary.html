Комментарии к полям списка
1.Поле номер-присваивается при добавлении нового сотрудника в список для выплаты, нумерация 1,2,3...n.
2.Поле счет заполняется при выборе сотрудника из справочника сотрудников.
3.Поле ФИО заполняется при выборе сотрудника из справочника сотрудников.
4.Поле ИНН заполняется при выборе сотрудника из справочника сотрудников.
5.Поле Сумма, грн. - поле для ручного заполнения. Допускаются только числовые значения поля. Возможно также групповое заполение поля Сумма, грн. для выделенных сотрудников. При выделении чекбоксом хотя бы одного сотрудника в списке, становится доступным поле Сумма на одного сотрудника. При заполенном поле Сумма на одного сотрудника становится доступной кнопка Проставить сумму. При нажатии кнопки Проставить сумму, содержимым поля Сумма на одного сотрудника заполняется каждое поле Сумма, грн. у выделенных сотрудников.


Поле Количество сотрудников отражает текущее количество сотрудников в списке
Поле Общая сумма отражает текущую сумму по сотрудника, которые есть в списке

После нажатия кнопки Добавить, открывается справочник сотрудников для выбора и добавления сотрудников в список

Кнопка Удалить выделенные записи доступна, если выбран хотя бы один чекбокс. При ее нажатии, выделенный сотрудник исключается из списка

При нажатии на кнопку Сохранить зарплатная ведомость сохраняется и становится доступной кнопка Сформировать платежи.

После нажатия кнопку Отменить осуществляется возврат на страницу зарплатных ведомостей без сохранения ЗП ведомости

При нажатии на кнопку формируется 2 платежа (если есть комиссия за перечисление) и 1 платеж (если такой комиссии нет). Платежи формируются в статусе Новый.

При нажатии на кнопку также появляется окно для выбора ЭЦП (ниже), до нажатия оно скрыто

Требования к формируемым платежам
1.Платеж на выплату ЗП. скриншот платежного поручения. Номер-по порядку, дата-текущая. Платежное получение формируется с реквизитами клиента, указанными в ЗП ведомости. Счетом получателя является счет 2924, открытый для клиента и указанный в договоре, как транзитный счет для выплаты ЗП. Остальные реквизиты получателя соответствуют реквизитам клиента. Сумма - сумма в ЗП ведомости. Назначение платежа формируется по шаблону: [Вид начисления][Название клиента][период начисления][номер ЗП ведомости][Дата ЗП ведомости]. Все данные из ЗП ведомости.
1.Платеж на оплату комиссии банку. скриншот платежного поручения. Номер-по порядку, дата-текущая. Платежное получение формируется с реквизитами клиента, указанными в ЗП ведомости. Счетом получателя является счет 2924 для оплаты комиссии и указанный в договоре, как счет для оплаты комиссии банку. Получатель-банк, остальные реквизиты соответствуют реквизитам клиента. Сумма - рассчитывается, исходя из суммы ЗП ведомости и комиссии. Назначение платежа формируется по шаблону: [Комиссия банку за зачисление средств на СКР сотрудников][Название клиента][Вид начисления][номер ЗП ведомости][Дата ЗП ведомости]. Все данные из ЗП ведомости.

Скриншот связанных платежей из БИФИТа

По ссылке пользователь может перейти на краткое представление сформированных платежей. Для удобства они представлены на отдельной странице

При нажатии на кнопку "Обзор" на экран выводится окно проводника для поиска файла ЭЦП. После выбора нужного файла, его название появляется в текстовом поле и становится активной кнопка "Применить".
При нажатии на нее инициируется процесс наложения ЭЦП на ЗП ведомость. После этого-возврат на страницу ЗП ведомостей.

Комментарии к странице, если она используется для копирования или редактирования ЗП ведомости
